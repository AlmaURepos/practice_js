# Архитектура проекта Task 2

## Цель
Продемонстрировать использование статических и динамических импортов, а также модульной архитектуры.

## Структура модулей

```
index.html          — точка входа, подключает app.mjs статически
modules/
  constants.mjs     — экспорт констант (appTitle)
  utils.mjs         — утилиты DOM (createElement)
  core.mjs          — бизнес-логика (класс TodoApp, default export)
  app.mjs           — инициализация приложения (статический импорт)
  advanced_feature.mjs — статистика, загружается динамически по клику
  lazy_component.mjs   — ленивый компонент, загружается при скролле
```

## Типы импортов

- **Статические**: `app.mjs` -> `core.mjs`, `utils.mjs`, `constants.mjs`
- **Динамические**: `advanced_feature.mjs` (по клику), `lazy_component.mjs` (IntersectionObserver)

## Принцип

Каждый модуль отвечает за одну задачу. Динамические импорты используются для отложенной загрузки модулей, которые не нужны сразу при старте.
