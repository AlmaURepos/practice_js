

## Реализованные паттерны

### 1. Singleton (SCENARIO_01) - Креационный
```javascript
class AppConfig {
  static getInstance() { ... }
}
```
Обеспечивает единственный экземпляр конфигурации приложения с глобальной точкой доступа. Используется для централизованного управления настройками без создания дублирующихся объектов.

### 2. Command (SCENARIO_05) - Поведенческий
```javascript
class MoveCardCommand { execute() { ... } undo() { ... } }
class ActionHistory { execute(command) { ... } }
```
Инкапсулирует действия (перемещение карточек, переименование) в объекты команд, позволяя реализовать функции undo/redo с сохранением истории операций.

## Ключевые выводы

1. **Диагностика паттерна по языку**: Ключевые слова в описании проблемы (единственный экземпляр, несовместимый интерфейс, оповещение, пошагово) напрямую указывают на правильный паттерн.

2. **Теория и практика**: Понимание абстрактных определений GoF позволяет быстро применять паттерны к конкретным проблемам разработки.

3. **Правильный выбор = Меньше проблем**: Использование правильного паттерна значительно упрощает код, повышает его гибкость и поддерживаемость.
