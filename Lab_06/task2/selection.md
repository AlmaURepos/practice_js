### SCENARIO_01
- Problem Summary: Приложение должно иметь одну единственную глобальную конфигурацию, к которой имеют доступ все компоненты, не создавая дублирующихся экземпляров.

- Selected Pattern Family: Creational

- Selected Specific Pattern: Singleton


### SCENARIO_02
- Problem Summary: Нужно интегрировать старую библиотеку с несовместимым интерфейсом в новую систему, не изменяя исходный код библиотеки.

- Selected Pattern Family: Structural

- Selected Specific Pattern: Adapter


### SCENARIO_03
- Problem Summary: Система должна оповещать множество подписчиков об изменении состояния события без прямого связывания между компонентами.

- Selected Pattern Family: Behavioral

- Selected Specific Pattern: Observer 

### SCENARIO_04
- Problem Summary: Необходимо сконструировать сложный объект (например, HTTP запрос или конфигурацию) пошагово с множеством опциональных параметров.

- Selected Pattern Family: Creational

- Selected Specific Pattern: Builder


### SCENARIO_05
- Problem Summary: Приложение для управления задачами должно позволять выполнять, отменять и переделывать операции над доской задач.

- Selected Pattern Family: Behavioral

- Selected Specific Pattern: Command


### SCENARIO_06
- Problem Summary: Приложение взаимодействует со сложной системой платежей с множеством процессов, требуется упростить интерфейс для клиентов.

- Selected Pattern Family: Structural

- Selected Specific Pattern: Facade
