### Классифицированные паттерны

1. **Singleton** — Креационный, гарантирует единственный экземпляр класса
2. **Observer** — Поведенческий, система подписки на события
3. **Facade** — Структурный, упрощение сложного интерфейса
4. **Factory Method** — Креационный, создание объектов различных типов
5. **Decorator** — Структурный, добавление функциональности к объекту
6. **Prototype** — Креационный, клонирование объектов
7. **Chain of Responsibility** — Поведенческий, передача запроса по цепи
8. **Command** — Поведенческий, инкапсуляция действий в объекты
9. **Flyweight** — Структурный, переиспользование объектов для экономии памяти
10. **Mediator** — Поведенческий, централизованное управление коммуникацией

### Ключевой вывод

Знание категории паттерна позволяет быстро выбрать правильное решение архитектурной задачи и избежать проектных ошибок.
