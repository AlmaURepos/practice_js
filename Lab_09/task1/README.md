В данной работе реализован паттерн Observer / Pub-Sub для управления подписками и событиями

Архитектура состоит из трех компонентов:
- EventBus (центральная шина событий) - хранит подписки и распространяет события
- NewsPublisher (издатель) - публикует новости по категориям и приоритетам
- Подписчики (EmailNotifier, PushNotifier, DashboardWidget) - получают и обрабатывают события

Логика работы: Publisher публикует событие через EventBus, все подписанные компоненты получают уведомления

Поддерживается:
- Подписка на специфичные категории новостей (sports, tech, business)
- Одноразовая подписка на urgent события (subscribeOnce)
- Динамическая подписка/отписка
- Обработка ошибок в обработчиках событий

Результат: гибкая система обработки событий с полной развязкой компонентов
