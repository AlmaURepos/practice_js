В данной работе реализованы паттерны Mediator и Command для управления умным домом

Архитектура состоит из четырех компонентов:
- HomeMediator (посредник) - координирует взаимодействие между устройствами
- Устройства (Light, Thermostat, SecuritySystem) - исполнители команд с состоянием
- CommandManager (менеджер команд) - управляет командами с поддержкой undo/redo
- Команды (TurnOnLightCommand, SetTemperatureCommand и т.д.) - инкапсулируют действия

Логика работы: Mediator регистрирует устройства, CommandManager выполняет команды, которые изменяют состояние через Mediator

Поддерживается:
- Управление тремя типами устройств (свет, термостат, охрана)
- История команд с полной поддержкой undo/redo
- Макрокоманды для составных операций
- Логирование всех операций через EventBus

Результат: централизованная система управления умным домом с полной историей действий
